BOYS OF WINTER LEAGUE — WHERE TO PUT YOUR DATA FILES
=====================================================

FHM BUNDLE MODE (Franchise Hockey Manager semicolon exports)
-------------------------------------------------------------
If this folder contains **team_data.csv**, the importer uses FHM layout automatically:
  - Delimiter is ";" (semicolon).
  - Expected files include: team_data.csv, player_master.csv, schedules.csv,
    team_records.csv, team_stats.csv, boxscore_*.csv, player_skater_stats_rs.csv,
    player_goalie_stats_rs.csv, draft_index.csv, draft_info.csv, player_contract.csv,
    league_data.csv, divisions.csv, player_ratings.csv, etc.
  - Team aggregate CSVs: **team_stats.csv** (regular season) and **team_stats_playoffs.csv** (playoffs), stored separately by segment.
  - Career register CSVs (optional; each is loaded if present):
      player_skater_career_stats_rs.csv   — active skater, regular season
      player_skater_career_stats_po.csv   — active skater, playoffs
      player_skater_retired_career_stats_rs.csv / _ps.csv / _po.csv — retired skater
      player_goalie_career_stats_rs.csv / _ps.csv / _po.csv — active goalie career
      player_goalie_retired_career_stats_rs.csv / _ps.csv / _po.csv — retired goalie

Only rows with League Id = 0 are imported for teams/schedules/standings (main league).

Optional: if **team_standings.csv** (comma CSV) is present in this folder, it is applied **after** the FHM import and overwrites standings for matching teams (by abbreviation, slug, fhm_team_id, or team name). Leave season columns blank to target the current season.

Optional: **season_label.txt** — one line; if present, used as the full season title on the site instead of deriving it from schedule dates. (Schedule dates in FHM exports are often not strict ISO-8601; the importer now tolerates that when computing the default label.)


After major schema updates, reset the database then re-import:
  python scripts\reset_db.py
  python run.py
  python scripts\import_data.py


1) FHM / GENERIC CSV EXPORTS (raw imports)
------------------------------------------
Place every CSV you want the importer to read in this exact folder:

  C:\Users\keeno\Projects\Boys-Of-Winter-League\data\imports\raw\

The pipeline reads UTF-8 (and auto-detects other encodings). Comma or semicolon is detected.

Legacy file names (comma CSVs) must match:

  teams.csv
  seasons.csv
  players.csv
  team_standings.csv
  games.csv
  player_skater_stats.csv
  player_goalie_stats.csv
  game_skater_stats.csv
  game_goalie_stats.csv
  scoring_events.csv
  penalty_events.csv
  prospects.csv
  drafts.csv
  draft_picks.csv
  history_awards.csv
  history_champions.csv

Missing files are skipped with a warning. Import order is fixed (teams first, then seasons, players, etc.).

Run the importer from the project root:

  python scripts\import_pipeline\runner.py

After import, the player search index (FTS) is rebuilt automatically.

Tip: Column headers are normalized (case/spaces). Use logical names such as:
  name, abbreviation, fhm_team_id, season_id, team_id, player_id, fhm_game_id, etc.
See scripts/import_pipeline/runner.py for aliases (e.g. abbr -> abbreviation).


2) TEAM LOGOS (12 teams)
------------------------
Put one image per team in this folder (not under data/imports):

  C:\Users\keeno\Projects\Boys-Of-Winter-League\app\static\logos\teams\

The file name must match the team’s **slug** exactly, plus an extension:

  .png  (preferred)
  .webp
  .jpg
  .svg

How to get the slug:
  - Open any team page: URL is /team/<slug> — use that <slug> as the file base name.
  - Legacy comma CSV (teams.csv): use the **slug** column if present; otherwise the importer builds a slug from name/abbreviation.
  - FHM import (team_data.csv): slugs are built as lowercase abbreviation + "-t" + FHM team id, e.g. abbr "ANC" and teamid 12 → **anc-t12.png**

Examples:
  frost.png           — only if that team’s slug is literally "frost"
  anc-t12.webp        — typical FHM-style slug

If no file matches the slug, the site uses logos/teams/placeholder.svg (the “BOWL” tile).

4) PLAYER HEADSHOTS (optional)
------------------------------
For future use / player pages:

  C:\Users\keeno\Projects\Boys-Of-Winter-League\app\static\players\

Image files should be named: firstname_lastname_day_month_year.png (all lowercase; date = birth
day, e.g. wayne_gretzky_26_1_1961.png). Lookup is case-insensitive. You may omit headshot_path
in players.csv when the file matches that pattern; imports set the expected path when birth_date
is present. Otherwise store headshot_path relative to static, e.g. players\wayne_gretzky_26_1_1961.png
